Calling the API from Native Apps

A "native" QuickSchools app is one that is developed using the App Developer Console and runs within the QuickSchools application. Contrast this with external apps running on a different site.

Native apps call the API using specialized javascript methods we've prepared for you. You do not need an API key to make the calls. Rather, we authenticate and authorize against the currently logged-in user who is using your app.

This does mean that there are restrictions on which api methods can be accssed by your app, based on who is using it. For example, if a parent or student is logged in, your app will get an authorization exception if it tries to make an API call to change grades.

If you'd like further information on building native QuickSchools apps, click here to see the documentation.

Making the API call from javascript

You can make GET, POST, PUT and DELETE calls to the API using either Rest.get(), Rest.post(), Rest.put() or Rest.del().

  • The first parameter is the API path.
  • The second parameter is a javascript object with the parameters that you want to send.
  • The third parameter is the context object for the callback.
  • The fourth parameter is the callback method.

The following examples illustrate this.

API Keys

If you want to make API calls from your school webserver or other school-specific application, you'll need an API key.

Remember that you shouldn't use an API Key to make calls from native QuickSchools apps that you build from the App Developer Console. See that section for details.

You create and manage API Keys from within your QuickSchools account. Be sure to keep those secure, and never expose them in javascript. It should only be stored and used within secure server environments.

Once you've gotten an API key, send that along with all API calls as the "apiKey" parameter.

The following are the steps to get an API Key.

Step 1: Give yourself access to the API Keys page.

Go to the Settings menu on the top. Scroll down till you see the section titled "Interested in developing your own apps?" Click Enable (or Configure if you have already enabled it in the past). Now give yourself access to the API Key page.

Step 2: Navigate to the API Keys page.

You'll see a new menu appear titled "App Developer Console". Click on that, and if you are not yet at the API Keys page, click on the API Keys submenu.

Step 3: Create an API Key.

Click on "Create new API Key". Enter in some notes so you can remember what this key is for. Once you've generated the API Key, you'll be able to show the secret key from that screen.

You're done! You can now send the API Key as the "apiKey" query parameter in your API calls.
